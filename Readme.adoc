= Brainfuck to zig compiler

Have you ever wanted to have performance in your brainfuck code? Look no further.

This build scripts accepts a brainfuck file as the environment variable `BF_FILE_PATH`, generates a zig file with it, and compiles it as `program` in your usual `zig-out/bin` output folder.

As an example, `helloworld.bf` is available. On linux, just run `BF_FILE_PATH=./helloworld.bf zig build` to compile and generate.

The generated zig file is available in the `generated` folder for reference.

== Options

The generated program can be tuned with build options:

[source,sh]
----
BF_FILE_PATH=./helloworld.bf zig build -Doptimize=ReleaseFast \
  -Dcell-bits=16 \      # cell width: 8 (default), 16, or 32
  -Dtape-size=30000 \   # number of cells (default 65535)
  -Dstdout-buffer=65536 # stdout buffer in bytes (default 65536)
----

`cell-bits` changes the arithmetic modulus of every cell (all wrapping is done
modulo `2^cell-bits`); `.` always writes the low 8 bits.

== Benchmarking

`bench/bench.sh` compiles a brainfuck program and times the resulting binary:

[source,sh]
----
bench/bench.sh                                  # default: bench/mandelbrot.bf, ReleaseFast
bench/bench.sh bench/mandelbrot.bf Debug        # compare optimization levels
bench/bench.sh bench/mandelbrot.bf ReleaseFast
----

The corpus includes:

* `bench/mandelbrot.bf` — Erik Bosman's mandelbrot viewer, a heavy, loop- and
  arithmetic-dominated workload.
* `bench/scan.bf` — a scan-dominated workload (`[>]` pointer scans), which the
  compiler lowers to a vectorized byte search.

When a `.sha256` file sits next to the program, the compiled binary's output is
verified against it before timing.
